# LAN Chat

一个面向局域网场景的纯前端聊天应用。用户在同一局域网内访问同一个页面地址后，可以创建临时房间、自动发现房间、直接加入聊天，并通过 WebRTC 数据通道发送文字和图片消息。

当前项目采用 **Vue 3 + TypeScript + Vite** 构建前端界面，配合一个轻量的 **Node.js + WebSocket** 信令节点做房间发现和加入流程协调。

## 项目特点

- **局域网可见房间列表**：所有设备访问同一个信令节点页面后，可以看到一致的在线房间。
- **WebRTC 点对点通信**：聊天数据通过 WebRTC DataChannel 传输，不依赖传统消息后端。
- **支持文字和图片消息**：可发送文本，也可选择图片或直接粘贴图片发送。
- **支持主题切换**：可以切换界面主色，并将选择保存到浏览器缓存中。
- **历史消息同步**：新加入成员可以接收已有聊天历史。
- **房主本地持久化**：消息历史保存在房主浏览器的 IndexedDB 中。
- **自动加入 + 手动邀请码加入**：既支持局域网内自动发现直连，也保留文本邀请码的配对方式。
- **移动端 / 桌面端通用界面**：界面已针对不同设备形态做适配。

## 当前架构

项目不是“纯静态页”模式，而是由两部分组成：

| 模块 | 作用 |
| --- | --- |
| `src/` | Vue 前端，负责房间创建、加入、聊天、图片传输、历史展示等 |
| `server/index.mjs` | 轻量信令节点，负责房间列表维护、加入请求转发、静态资源服务 |

当前通信模型：

1. 房主进入房间后，会把房间状态注册到局域网信令节点。
2. 其他设备访问同一个页面地址后，可看到在线房间列表。
3. 访客可以直接请求加入，或通过邀请码手动完成配对。
4. 成功连接后，聊天消息通过 WebRTC 数据通道传输。
5. 当前实现中，房主承担房间生命周期、历史同步和消息转发职责。

## 本地开发

建议使用 **pnpm**。

```bash
pnpm install
pnpm dev
```

`pnpm dev` 会同时启动：

- `pnpm dev:web`：Vite 前端开发服务器，默认端口 `5173`
- `pnpm dev:signal`：局域网信令节点，默认端口 `5174`

开发模式下，Vite 会把 `/ws` WebSocket 请求代理到本地信令节点。

如果你希望只单独启动某一部分，也可以使用：

```bash
pnpm dev:web
pnpm dev:signal
```

## 生产构建与运行

```bash
pnpm build
pnpm serve
```

- `pnpm build`：执行 `vue-tsc --noEmit` 和 Vite 打包
- `pnpm serve`：启动 Node 服务，对外提供 `dist/` 静态资源和 `/ws` 信令入口

默认情况下：

- 页面服务端口：`5173`
- 仅信令模式端口：`5174`

也可以通过环境变量覆盖端口：

```bash
PORT=8080 pnpm serve
```

## 使用方式

### 房主

1. 打开页面并填写昵称
2. 创建临时房间
3. 将当前页面地址发给同一局域网内的其他设备
4. 其他设备访问后即可看到房间，并直接加入

### 访客

1. 访问房主提供的页面地址
2. 在“当前可见房间”中选择房间直接进入
3. 如果自动加入不可用，也可以使用邀请码文本手动加入

## 当前边界与限制

- **无公网后端**
- **无 TURN 服务**
- **无公共 STUN 依赖**
- 设计目标是同一局域网内设备之间使用
- 房主离开后，房间与其维护的历史也会结束
- 历史消息持久化当前存放在房主浏览器本地，而不是独立数据库

## 适用场景

- 家庭或办公室内的临时多设备聊天
- 同一 Wi-Fi 下的轻量协作沟通
- 无需注册、无需公网部署的本地即时通信实验
